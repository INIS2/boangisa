💡 단답형 정답
1번 정답 (시스템 - 윈도우 계정 그룹)

정답:

(A): Power Users
(B): Backup Operators
(C): Users
해설: 윈도우 OS의 기본 보안 그룹 구조는 권한 분리 원칙에 따라 설계되었습니다. Power Users는 일반 사용자보다 많은 권한을 가지나 관리자 권한에는 제한이 있으며, Backup Operators는 백업 목적으로 모든 파일에 접근할 수 있는 특수 권한을 가집니다. Users 그룹은 일반적인 사용자 작업에 필요한 기본 권한을 제공합니다.

실무 연결: 실제 기업 환경에서는 최소 권한 원칙에 따라 사용자별로 적절한 그룹을 할당하여 보안을 강화합니다.

2번 정답 (네트워크 - 전자파도청)

정답: TEMPEST (Telecommunications Electronics Material Protected from Emanating Spurious Transmissions)

해설: TEMPEST는 전자기기에서 발생하는 불필요한 전자파 방사를 통한 정보 유출을 방지하기 위한 보안 기술입니다. 정부, 군사 및 민간 기관에서 전자기 방사로 인한 도청 및 정보 유출의 위험을 최소화하기 위해 차폐, 필터링, 거리 유지, 장비 인증 등의 방법을 사용합니다.

실무 연결: 금융기관이나 정부기관에서는 TEMPEST Zone 설정을 통해 민감한 정보 처리 공간을 보호합니다.

3번 정답 (네트워크 - IPSec)

정답: 접근제어, 비연결형 무결성, 데이터 근원지 인증, 재전송 방지, 기밀성, 제한적 트래픽 흐름의 기밀성

해설: IPSec은 네트워크 계층에서 포괄적인 보안 서비스를 제공하는 프로토콜 스위트입니다. 인증 헤더(AH)와 캡슐화 보안 페이로드(ESP)를 통해 데이터의 기밀성, 무결성, 인증을 보장하며, VPN 구축의 핵심 기술로 활용됩니다.

실무 연결: 기업 원격 근무 환경에서 Site-to-Site VPN 구축 시 IPSec이 필수적으로 사용됩니다.

4번 정답 (애플리케이션 - 세션 하이재킹)

정답: 세션 하이재킹(Session Hijacking)

해설: 세션 하이재킹은 활성화된 네트워크 세션을 공격자가 탈취하여 정당한 사용자인 것처럼 서버와 통신하는 공격입니다. 세션 ID 추측, 세션 고정, 중간자 공격 등의 방법으로 수행되며, HTTPS 사용, 세션 ID 무작위화, 세션 타임아웃 설정 등으로 방어할 수 있습니다.

실무 연결: 웹 애플리케이션 개발 시 JWT 토큰 기반 인증과 함께 세션 관리 보안이 중요합니다.

5번 정답 (정보보호관리 - 보안통제유형)

정답:

(A): 예방 통제
(B): 물리적 접근 통제
(C): 논리적 접근 통제
해설: 정보보호 통제는 사전 예방과 사후 대응의 관점에서 분류됩니다. 예방 통제는 보안 사고의 발생을 미연에 방지하는 능동적 보안 대책이며, 물리적 접근 통제는 시설 및 장비에 대한 물리적 접근을 제한하고, 논리적 접근 통제는 시스템 및 데이터에 대한 전자적 접근을 통제합니다.

실무 연결: 데이터센터 운영 시 생체인식 출입통제(물리적)와 다중인증(논리적)을 동시에 적용합니다.

6번 정답 (애플리케이션 - 메일보안)

정답:

(A): RELAY
(B): DISCARD
해설: 메일 서버의 릴레이 설정은 스팸 메일 차단과 정상 메일 전송을 위해 중요합니다. RELAY는 메일 중계를 허용하며, DISCARD는 메일을 삭제 처리합니다. REJECT는 발신자에게 오류 메시지를 보내지만, DISCARD는 조용히 삭제하여 스팸 발송자에게 정보를 제공하지 않습니다.

실무 연결: 기업 메일 서버 운영 시 도메인별 정책 설정으로 스팸을 효과적으로 차단할 수 있습니다.

7번 정답 (애플리케이션 - 웹 프락시)

정답: 웹 프락시(Web Proxy)

해설: 웹 프락시는 클라이언트와 서버 간의 HTTP/HTTPS 통신을 중개하여 트래픽을 분석하고 조작할 수 있는 도구입니다. 보안 테스팅에서는 취약점 발견과 공격 시뮬레이션에 사용되며, Burp Suite, OWASP ZAP, Paros 등이 대표적인 도구입니다.

실무 연결: 모의해킹 수행 시 웹 애플리케이션의 HTTP 요청/응답을 분석하여 SQL Injection, XSS 등의 취약점을 발견합니다.

8번 정답 (네트워크 - 서브넷마스크)

정답: 11111111.11111111.11111111.11000000

해설: 서브넷 마스크 255.255.255.192는 /26 네트워크를 의미합니다. 각 옥텟을 2진수로 변환하면 255는 11111111, 192는 11000000이 됩니다. 이는 호스트 비트 6개로 62개의 호스트 주소를 사용할 수 있음을 나타냅니다.

실무 연결: 네트워크 설계 시 서브네팅을 통해 브로드캐스트 도메인을 분리하고 네트워크 성능을 향상시킵니다.

9번 정답 (애플리케이션 - SW테스트기법)

정답: Fuzzing(퍼징)

문제 14번 (네트워크 - 스니핑탐지기법)

정답:

Ping을 이용한 스니핑 탐지 방법:

스니핑이 의심스러운 호스트에 네트워크에 존재하지 않은 MAC주소로 위조된 PING 메시지(ICMP Echo Request)를 보냅니다. ICMP Echo Reply가 돌아온다면, 해당 호스트는 무차별 모드(Promiscuous Mode)로 스니핑 중인 것으로 판단할 수 있습니다.

출제 의도: 네트워크 스니핑의 동작 원리와 탐지 방법에 대한 실무적 이해를 평가

실무 적용:

정상적인 네트워크 카드는 자신의 MAC 주소가 아닌 프레임은 무시합니다.
스니핑 도구는 무차별 모드에서 모든 네트워크 트래픽을 수집하기 위해 모든 프레임을 처리합니다.
위조된 MAC 주소로 전송된 패킷에 응답한다면 무차별 모드로 동작 중임을 의미합니다.
scapy와 같은 도구를 사용하여 이더넷 헤더의 MAC 주소를 위조할 수 있습니다.
학습 포인트: 네트워크 보안 관제 시 비정상적인 트래픽 패턴 탐지가 중요합니다.

문제 15번 (정보보호관리 - 기밀성등급)

정답:

기밀성 등급 정의:

(A) M(중) 등급: 기밀성이 중간 정도인 민감 정보를 저장/처리하므로, 업무 담당자 및 관리자 등 허가된 직원만 접근 가능(예: 인사 정보, 내부 재무 정보 등)

(B) L(하) 등급: 기밀성이 낮은 민감하지 않은 정보를 저장/처리하므로, 내부의 일반 직원도 접근 가능하며, 공개된 정보인 경우 일반 대중도 접근 가능(예: 마케팅 자료, 공시 정보, 웹사이트 콘텐츠 등)

출제 의도: 정보 자산의 기밀성 등급 분류 체계와 접근 권한 관리를 이해하고 있는지 평가

실무 적용:

기밀성 등급은 조직의 특성에 따라 다양하게 정의될 수 있습니다.
일반적으로 극비(Restricted), 기밀(Confidential), 내부(Internal), 공개(Public)로 분류하기도 합니다.
대규모 기업에서는 5단계(Very High, High, Medium, Low, Very Low)로 세분화하기도 합니다.
중요한 것은 조직의 정보 자산에 적합한 등급 체계를 수립하고 일관성 있게 적용하는 것입니다.
학습 포인트: ISMS-P 인증 시 정보자산 분류 및 관리 절차가 필수적으로 요구됩니다.

문제 16번 (네트워크 - IDS 행위)

정답:

IDS가 공격 탐지 시 수행할 수 있는 행위:

알림(Alert): 이메일, SMS, 관리 콘솔 내 메시지 출력 등을 통하여 보안 관제 담당자에게 알림
로깅(Logging): 탐지된 침입 시도를 로그로 남김(이후 분석 및 포렌식 조사를 위한 기초 자료로 사용)
세션 차단(Session Termination): 공격이 진행 중인 세션을 강제로 종료하여 침입을 차단(IPS 기능이 포함된 IDS의 경우)
시스템 연동(System Integration): 다른 시스템(예: SIEM, 방화벽, 스위치, 라우터)와 통합되어 효과적인 대응 수행(예: 종합 분석, 차단 정책 설정 등)
출제 의도: IDS의 능동적 대응 기능과 보안 시스템 간의 연계를 이해하고 있는지 평가

실무 적용:

IDS는 탐지(Detection) 기능이 주목적이지만, 현대의 IDS는 대응(Response) 기능도 포함합니다.
SNORT의 Action 유형(Alert, Log, Pass, Activate, Dynamic, Drop, Reject, SDrop)을 참고할 수 있습니다.
네트워크 기반 IDS와 호스트 기반 IDS는 서로 다른 대응 방식을 가집니다.
학습 포인트: 통합 보안 관제 환경에서 각 보안 솔루션의 역할과 연계 방안을 이해해야 합니다.

문제 17번 (네트워크 - ARP 리다이렉트)

정답:

ARP 테이블 조회 명령: arp -a
공격 유형: ARP Redirect (192.168.100.1이 Gateway라고 가정). 만일 해당 IP가 Gateway가 아니라면 ARP Spoofing
공격 판단 이유: Gateway에 해당하는 IP(192.168.100.1)의 MAC 주소가 192.168.100.5를 사용하는 PC의 MAC주소로 변조되었기 때문. 이 경우 Gateway로 향하는 모든 패킷은 192.168.100.5를 경유하게 되어, 메시지 내용 훔쳐보기, 메시지 위변조 등 공격이 가능하게 됨
대응 명령어: arp -s 192.168.100.1 00-0a-00-62-c6-09
출제 의도: ARP 프로토콜의 취약점과 실제 네트워크 환경에서의 대응 방안을 평가

실무 적용:

ARP Spoofing은 희생자 IP의 MAC를 변조하는 일반적인 공격을 지칭합니다.
Gateway 주소를 변조한 경우 정확한 공격명은 ARP Redirect가 맞습니다.
arp -s 명령어는 정적 ARP 엔트리를 생성하여 ARP 공격을 방어합니다.
네트워크 스위치의 포트 보안 기능이나 동적 ARP 검사(DAI) 기능으로도 방어할 수 있습니다.
학습 포인트:

중간자 공격(MITM)의 대표적인 사례로, 실제 네트워크 관리에서 자주 발생하는 보안 위협입니다.
오답 사례: arp -d는 삭제 명령으로 대응책이 될 수 없습니다.
문제 18번 (시스템 - 악성코드 분석)

정답:

PE 파일 악성코드 분석 방법:

1) 자동화 분석:
악성 파일 분석 과정에서 수작업을 최소화하고 효율성을 극대화하기 위해 자동화된 도구와 기술을 사용하는 방법

정적 분석 도구: PEiD, IDA Pro, YARA와 같은 도구를 이용하면 파일 헤더, 섹션, 코드 패턴 등을 자동으로 분석하여 악성 여부 판별이 가능
동적 분석 도구: Cuckoo Sandbox, Any.Run과 같은 도구를 이용하면, 악성 파일을 격리된 환경에서 실행하여 파일의 행위, 네트워크 활동, 시스템 변경 사항을 자동으로 기록하고 분석 가능
2) 반자동화 분석:
자동화된 도구를 사용하여 초기 분석을 수행하고, 분석가가 추가적으로 세부 사항을 수동으로 조사하여 정확성을 높이는 방법

자동화 도구로 파일을 격리된 환경에서 실행하고 초기 행동 분석을 자동으로 수행
분석가는 이 결과를 바탕으로 추가 조사 대상을 선정
디스어셈블러(예: IDA Pro, Ghidra)를 사용하여 자동화 도구가 식별한 의심스러운 코드나 행동을 중심으로 코드의 특정 부분을 수동으로 분석
3) 수동 분석:
주로 분석가의 전문 지식과 경험을 바탕으로 파일을 직접 분석하는 방법. 깊이 있는 조사와 상세한 이해를 필요로 하는 복잡한 악성코드 분석에 유용

초기 조사: 파일의 기본 속성(크기, 해시 값, 타임스탬프 등)을 확인하고, PE 헤더 등을 분석하여 실행 파일인지 확인
정적 분석: PE 파일의 DOS 헤더, PE 헤더, 섹션 헤더 등을 검사하여 파일 구조를 이해하고, IDA Pro, Ghidra, Radare2 등의 디스어셈블러를 사용하여 기계어 코드를 어셈블리 코드로 변환하고 분석
동적 분석: OllyDbg, x64dbg 같은 디버거를 사용하여 악성코드를 단계별로 실행하면서 코드의 동작을 추적하고 특정 행동을 유발하는 코드 부분을 식별
출제 의도: 악성코드 분석의 체계적인 접근 방법과 각 방법의 특징을 이해하고 있는지 평가

실무 적용:

PE 파일 포맷에 대한 이해는 윈도우 환경에서의 악성코드 분석에 필수적입니다.
각 분석 방법은 상호 보완적으로 사용되어 종합적인 분석 결과를 도출합니다.
자동화 분석으로 전체적인 개요를 파악하고, 수동 분석으로 세부적인 동작 원리를 분석하는 것이 효과적입니다.
학습 포인트:

실제 SOC 운영 시 1차적으로 자동화 분석을 통해 위험도를 판단하고, 고위험군에 대해서는 수동 분석을 병행합니다.
Sandbox 환경 구축 시 가상화 탐지 우회 기술도 고려해야 합니다.
